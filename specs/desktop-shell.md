# Desktop Shell

## Auto-update visibility

- Packaged macOS desktop builds show a compact update pill in the titlebar lane when an auto-update download is in progress.
- The pill stays out of the way of the native traffic lights and does not cover primary editor controls.
- While downloading, the pill clearly says that an update is in progress and shows download progress as a percentage.
- After the download finishes, the pill switches to an update-ready state with a restart action so the user can install the update later without reopening a modal.
- The update pill is not shown in the web app or when the desktop app is idle with no active update download.
